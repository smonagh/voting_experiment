The procedure on your computer screen

In each round, the first screen, shown below, informs you that you are Type A participant and that your subject number is A1.

 

You are now participant A1

The players in your group have the following number of votes
Type A Participant A1 (you) A2 A3
Number of Votes 3 1 1

The sum of the votes is 5 .

The project with at least 3 votes will be recommended to participant B,

 

The screen further informs you about the number of votes each Type A participant holds and the number of votes that are required to recommend a project to the Type B participant. This example screen informs you that you have two votes, the sum of the votes of all participants is 5, and that the project that receives at least 3 votes will are recommended to participant B.

The other two Type A participants are A2 and A3. They will seem a similar screen as you, letting them know about their subject number and how many votes they have. Thus, for example, A3 will see the identical screen as you do, with the only exception that that participant is told that he or she has the subject identifier A3. Following the example from the screen above, A3 will learn from his or her screen that she or she has one vote.

To move on to the next screen, click on “continue”.

In each round, the second screen informs all Type A participants, including you, you about your earnings for Type A and Type B participants when either Project X or Project Y is implemented. All participants of Type A are informed about the earnings associated with both projects at the same time.

Below is an example screen containing the earnings about Projects X and Y.

This screen informs you that if project X is chosen to be implemented by Type B, each of the Type A participants will earn E$16 and the Type B participant will earn E$4.

This screen further informs you that if project X is chosen to be implemented by Type B, each of the Type A participants will earn E$4 and the Type B participant will earn E$16.

Recall, at the end of the decision making round, one of the three Type A partisans will be randomly chosen to receive the earnings.

As noted previously, Type B participant has no information about the earnings associated with either of the two projects. Type B participants are also not informed about the procedure (total number of votes, votes assigned to each voter, the 50% majority of votes required for a message to be sent to Type B) by which the group of Type A participants decides on the message to be sent.

To move on to the next screen, click on “continue”.

In each round, the third screen is the decision screen. Here, you will vote on whether to send Message X or Message Y to the Type B participant.

On this decision screen, you are reminded in the upper row whether you are person A1, A2 or A3, and about the number of votes of each participant.

At the bottom of this screen you are reminded how many votes are needed for the majority rule procedure to send either Message X or Message Y to the Type B participant.

To vote whether to send Message X or Message Y to participant B, use the computer mouse to click on that red field which indicates the message which you would like to vote for sending to Type B. your choice. The upper field is for a vote in favor of sending Message X to participant B. The lower field is a vote in favor of sending Message Y to participant B.

The project with at least 4 votes will be recommended to participant B

   

After you have clicked the button for either sending Message X or sending Message Y, you will advance automatically to the next round , where you may be allocated a new identification number, (A1, A2, or A3). In this new round, you and the other two participants may have different number of votes than in the previous round.
